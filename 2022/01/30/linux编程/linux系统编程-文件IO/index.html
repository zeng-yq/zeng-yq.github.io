manpage介绍

使用 man 命令，查看的就是 manpage，man 就是 manual（手册）的缩写。man page 按照文档主题的不同，分成9个部分。

  1. Executable programs or shell commands
  2. System calls
  3. Library calls (functions within program libraries)
  4. Special files (usually found in /dev)
  5. File formats and conventions (eg /etc/passwd)
  6. Games
  7. Miscellaneous (including macro packages and conventions)
  8. System administration commands (usually only for root)
  9. Kernel routines

open函数

打开或者创建一个文件

  • 函数定义

  • 返回值

    如果操作成功，返回一个文件描述符，如果操作失败，返回-1。

  • 参数

    • pathname

      想要打开的文件路径名

    • flags

      打开文件所采用的操作；

      必须常量，且只允许指定一个：O_RDONLY（只读）、O_WRONLY（只写）、O_RDW（读写）；

      可选常量：O_APPEND、O_CREAT、O_TRUNC......

    • mode

      mode参数表示设置文件访问权限的初始值，和用户掩码umask有关，比如0644表示-rw-r-–r–-。第三个参数是在第二个参数中有 O_CREAT 时才起作用，如果没有，则第三个参数可以忽略。

read/write函数

读/写文件

read函数

  • 函数定义

    read() attempts to read up to count bytes from file descriptor fd into the buffer starting at buf.

  • 返回值

    • 0：读到文件末尾。
    • 成功：（ > 0 ）读到的字节数。
    • 失败： -1， 设置 errno = -1； 并且 errno = EAGIN 或 EWOULDBLOCK， 说明不是 read 失败，而是 read 在以非阻塞方式读一个设备文件或网络文件，并且文件无数据（因此，返回 -1 时不能简单认为是读失败。参考下面阻塞和非阻塞部分）。

  • 参数

    • fd：文件描述符

      STDIN_FILENO：0

      STDOUT_FILENO：1

      STDERR_FILENO：2

    • buf：存数据的缓冲区

    • count：缓冲区大小

write函数

  • 函数定义

    write() writes up to count bytes from the buffer pointed buf to the file referred to by the file descriptor fd.

  • 返回值

    • 成功： 写入的字节数
    • 失败： -1， 设置 errno

  • 参数

    • fd：文件描述符

    • buf：待写出数据的缓冲区

    • count：数据大小

文件描述符

PCB（进程控制块）的本质是一个结构体，其中一个成员是文件描述符表。

文件描述符是指向一个文件结构体的指针。

文件描述符：0/1/2/3/4....../1023 ，一个进程只能打开1024个文件。
