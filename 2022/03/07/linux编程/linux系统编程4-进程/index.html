用户权限及组权限

liunx中有文件有三种权限，用户权限，群组权限，其他用户权限，分别为 r(可写)、w(可读)、x(可执行) 。

还有 s/t(特殊权限)，有如下三种情况：

  1. rwsrwxrwx 用户权限出现 s，替代了 x
  2. rwxrwsrwx 群组权限出现 s，替代了 x
  3. rwxrwxrwt 其他权限出现 t，替代了 x

如果一个可执行文件有 u+S 权限，当别的用户在调用该可执行文件的时候，则会切换到该文件所属用户的身份来执行。

如果一个可执行文件有 g+S 权限，当别的用户在调用该可执行文件的时候，则会切换到该文件所组用户的身份来执行。

真实/有效用户id

  • 真实用户id（UID）

    该ID表明当前运行程序的执行者

  • 有效用户id（EUID）

    表示该ID是程序的所有者

  • 例如：

    程序 test 的所有者为 101/apple，

    以 101 运行该程序时 UID 和 EUID 都是 101，

    但由于中间要访问某些系统资源，需要使用 root 身份，

    此时 UID 为 0，而 EUID 仍是 101。

  • getuid/geteuid

  • getgid/getegid

  • setuid/setgid/seteuid/setegid

  • setreuid/setregid
